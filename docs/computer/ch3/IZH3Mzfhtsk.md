# IPv4 and IPv6 — Class 9 Computer Science Lecture Summary

## Lecture Overview
This lecture zeroes in on Layer 3 of the OSI model (The Network Layer). The teacher explains IP Addresses, the unique digital address required for any device to connect to the internet.

## Main Concepts Explained by the Teacher
The instructor explains that just like houses need a postal address, computers need an Internet Protocol (IP) address. Sir introduces IPv4, the older standard that uses 32-bit addresses (e.g., 192.168.1.1). Because the world ran out of IPv4 addresses due to the explosion of smartphones, the teacher explains the transition to IPv6, which uses massive 128-bit addresses, ensuring we never run out again.

## Important Definitions and Terms
- **IP Address:** A unique string of numbers separated by periods (or colons) that identifies each computer using the Internet Protocol.
- **IPv4:** The fourth version of IP, using 32-bit addresses.
- **IPv6:** The newest version of IP, using 128-bit hexadecimal addresses.

## Key Points and Lists from the Lecture
- IPv4 format: Four numbers separated by dots (e.g., 172.16.254.1).
- IPv4 can only hold about 4.3 billion unique addresses.
- IPv6 format: Eight groups of hexadecimal numbers separated by colons.
- IPv6 can hold $3.4 \times 10^{38}$ addresses (virtually infinite).

## What Students Should Remember
Pro tip: The bit-size difference is the most frequently asked MCQ. You guys must memorize that IPv4 is strictly 32-bit, and IPv6 is strictly 128-bit. Furthermore, knowing that we ran out of IPv4 addresses is the key to answering "why" IPv6 was created.
